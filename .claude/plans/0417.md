# Replace acpx with @anthropic-ai/claude-agent-sdk

## Context

当前 `src/subagent/acpx-backend.ts` 通过 `npx acpx` 或 `claude -p` 启动子进程,再用 JSON-RPC / stream-json 解析输出。这层中间件不稳定且偏重(一条 prompt → 进程启动 → CLI 初始化 → ACP 协议解析)。参考 vibe-anywhere PR #143 / #145 的做法,直接用 `@anthropic-ai/claude-agent-sdk` 的 `query()` 调 Claude,砍掉进程和协议两层。

MVP 范围: 只支持 `claude` 这一个 subagent。Copilot CLI 放 vNext,通过保留 `agent` 枚举字段占位。

## Design decisions (已确认)

1. SDK: `@anthropic-ai/claude-agent-sdk`
2. 命名: `AcpxBackend` → `SubagentBackend`,`AcpxEvent` → `SubagentEvent`,`ACPX_AGENTS` → `SUBAGENT_AGENTS`,文件 `src/subagent/acpx-backend.ts` → `src/subagent/backend.ts`
3. Agent 枚举保留 (`"claude"` only in MVP,为 vNext 的 copilot 占位)
4. 权限模式保留 `skip | allowlist | default` 三档,**配置键名直接复用 SDK 的 `allowedTools`**(不发明新词);映射规则:
   - `skip` → SDK `permissionMode: "bypassPermissions"`
   - `allowlist` → SDK `permissionMode: "default"` + `allowedTools: [...]`
   - `default` → SDK `permissionMode: "default"` (无 allowedTools)

## Key SDK usage reference (from vibe-anywhere claude-provider.ts)

```ts
import { query } from "@anthropic-ai/claude-agent-sdk";

const abortController = new AbortController();
const iterator = query({
  prompt,  // string, 或 AsyncIterable<SDKUserMessage>
  options: {
    cwd,
    model,
    maxTurns,
    permissionMode,   // "default" | "bypassPermissions" | ...
    allowedTools,     // string[]
    abortController,
    includePartialMessages: true,
  },
});

for await (const msg of iterator) {
  const ev = mapSdkMessage(msg);
  if (ev) yield ev;
}
```

Cancellation: `abortController.abort()`,`for await` 会抛 `AbortError`,捕获后 emit `{type:"error"}` + `{type:"done", exitCode:1}`。
